    Introduction

    Population Management is an API that lists locations and the total number of residents in each location broken down by gender. API users can perform CRUD operations on locations

    Development

    The application was developed with NodeJs and Express for routing. Postgres is the database used and sequelize is the ORM used.

    Installation

    Follow the steps below to setup a local development environment. First ensure you have Postgresql installed, and a version of Node.js equal or greater than v6.8.0.

    1. Clone the repository from a terminal git clone https://github.com/joel-ace/population-management.git.
    2. Change into the mock-sms-api directory cd population-management
    3. Rename .env_sample to .env and add the required DATABASE URL.
    4. Install project dependencies npm install
    5. Start the express development server npm run dev.

    API Summary

    EndPoint Functionality
    POST /api/v1/locations/ Creates a new location.
    GET /api/v1/locations/ Gets all locations.
    GET /api/v1/locations/:id Gets location using it's id.
    PUT /api/v1/locations/:id Updates a location using it's id.
    DELETE /api/v1/locations/:id Deletes location

    Locations

    Create Location

    Create a new location

    Request:

    {
    "name": "Houston",
    "malePopulation": 1302,
    "femalePopulation": 1908,
    "parentId": 89
}

    Response:

    {
    "location": {
    "id": 690,
    "name": "Houston",
    "malePopulation": 1302,
    "femalePopulation": 1908,
    "parentId": 89,
    "totalPopulation": 3210,
    "updatedAt": "2019-01-17T21:26:28.204Z",
    "createdAt": "2019-01-17T21:26:28.204Z"
}

    Request

    • Endpoint: POST: /api/v1/locations
    • Optional Field: parentId
    • Body (application/json)

    Response

    • Status: 201: Created
    • Body (application/json)

    Errors

    If you are new to HTTP status codes, you can use this as a reference to know what the returned error status codes used in Population Management API means.

    Population Management API uses the following code error codes:

    Error Code Meaning
    400 Bad Request -- You're sending a request with bad/not properly formatted parameters
    404 Not Found -- What you requested for could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
